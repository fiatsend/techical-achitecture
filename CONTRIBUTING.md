## Contributing

Thanks for taking a look.

### What’s useful to contribute here

- Clarifying edits that improve readability
- Diagrams (Mermaid preferred for easy rendering on GitHub)
- Risk analysis (security, compliance, operational)
- “Open questions” + proposed resolutions
- Links to canonical Stellar/Soroban specifications and docs

### What not to include

- Secrets or credentials (API keys, private keys, seed phrases, `.env` files)
- User PII or production identifiers

### Suggested PR checklist

- The change is scoped and explained (what + why)
- If adding a new component/flow, include a short sequence diagram
- Verify links resolve (SEPs, Anchor Platform, Soroban)
